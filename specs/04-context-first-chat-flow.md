# Spec 04: Context-First Chat Flow (Single User)

## Цель

Сделать Telegram-бота основным хранилищем контекста для одного пользователя, где обычный чат идет через LLM, а интересы/критерии влияют только на анализ YouTube.

## Область изменений

- `bot/src/index.ts`
- `bot/src/utils.ts`
- `bot/src/types.ts` (при необходимости)

## Пошаговый план

### 4.1 Единый контекст чата

- Использовать single-user сессию в памяти процесса.
- Хранить в ней:
  - историю диалога (`history`);
  - интересы (`interests`);
  - критерии (`criteria`).

### 4.2 Правила маршрутизации сообщений

- Отправлять в LLM:
  - любое обычное текстовое сообщение;
  - `/analyze <youtube_url>`;
  - сообщение с YouTube URL без команды.
- Не отправлять в LLM:
  - `/interests`;
  - `/criteria`;
  - `/clear`;
  - сервисные команды `/start`, `/help`.

### 4.3 Контекст для разных типов запросов

- Для обычного чата передавать в prompt историю диалога (ограниченное окно).
- Для видео-анализа передавать `criteria` и `interests`.
- Для обычного чата не передавать `criteria` и `interests`.

### 4.4 Очистка состояния

- `/clear` очищает:
  - историю диалога;
  - интересы;
  - критерии.

## Критерии приемки

- [ ] Обычный текст всегда уходит в LLM и учитывает историю.
- [ ] `/analyze` и авто-детект YouTube запускают анализ.
- [ ] `interests/criteria` влияют только на видео-анализ.
- [ ] После `/clear` старый контекст не используется.
