# Spec 09: Chat History Window (Local Buffer)

## Цель

Ограничить локальную историю бота и предотвратить разрастание контекста.

## Правила

- История хранится во временном массиве в памяти процесса.
- Массив содержит не более `10` элементов суммарно (`User/Assistant` вместе).
- Новые элементы добавляются в конец.
- При превышении лимита старые элементы удаляются с начала (FIFO).
- При видео-запросе история полностью очищается.

## Примечание

- История нужна для локального контекста и диагностики состояния бота.
- Основной контекст текстового диалога хранится в текущей `chat`-сессии OpenCode.

## Критерии приемки

- [ ] Лимит `10` строго соблюдается.
- [ ] При видео-запросе массив истории становится пустым.
