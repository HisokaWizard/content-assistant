# Spec 03: Интеграция и Тестирование

## Цель

Протестировать полное flow от отправки YouTube ссылки до получения summary.

## Шаги

### 3.1 Локальный Запуск Opencode Agent

- Запустить opencode agent в режиме сервера
- Проверить, что endpoint доступен
- Документировать адрес и порт

### 3.2 Локальный Запуск Telegram Бота

- Запустить Telegram-бота локально
- Настроить webhook или polling mode
- Убедиться, что бот онлайн

### 3.3 Тест Flow: Анализ YouTube Видео

- Отправить боту YouTube ссылку
- Дождаться обработки (может занять 10-30 секунд)
- Проверить получение:
  - Название видео
  - Summary (2-3 предложения)
  - Тезисы (3-5 пунктов)
  - Рекомендация (смотреть/не смотреть)

### 3.4 Тест Flow: Перевод

- Найти видео на английском языке
- Проверить, что summary на русском

### 3.5 Тест Flow: Сохранение Состояния

- Задать интересы (`/interests crypto, AI`)
- Задать критерии (`/criteria короткие видео, полезные факты`)
- Проанализировать видео
- Проверить, что критерии учитываются в рекомендации

### 3.6 Тест Flow: Очистка Сессии

- Выполнить `/clear`
- Проверить, что история очищена

### 3.7 Edge Cases

- Видео без transcript — проверить обработку ошибки
- Невалидная ссылка — проверить валидацию
- Пустое сообщение — проверить поведение

### 3.8 Документирование

- Зафиксировать успешные и неуспешные сценарии
- Определить базовые лимиты (количество запросов в день)
- Создать README с инструкцией запуска

## Критерии Приемки

- [ ] Полный flow работает: ссылка → summary → ответ в Telegram
- [ ] Перевод работает для английских видео
- [ ] Состояние сохраняется между запросами
- [ ] `/clear` очищает сессию
- [ ] Обработка ошибок корректная
- [ ] Система запускается за 10-15 минут

## Зависимости

- Работающий opencode agent (spec 01)
- Работающий Telegram бот (spec 02)
