# Spec 06: Hardening и Проверка Качества MVP

## Цель

Снизить риск регрессий и закрепить предсказуемое поведение бота после перехода на context-first flow.

## Пошаговый план

### 6.1 Устойчивость Telegram-ответов

- Исключить падения на Telegram parse mode:
  - либо отправлять plain text;
  - либо безопасно экранировать Markdown.
- Для длинных ответов предусмотреть разбиение на части при превышении лимитов Telegram.

### 6.2 Согласованность конфигурации Opencode

- Зафиксировать единый `OPENCODE_URL` во всех модулях.
- Явно задокументировать ожидаемый endpoint для чат-запросов.
- Проверить, что dev/prod запуск не расходится по портам.

### 6.3 Контракт prompt-формирования

- Формализовать шаблоны:
  - чат-запроса;
  - запроса на анализ видео.
- Добавить smoke-тесты на:
  - передачу истории только в чат;
  - передачу interests/criteria только в анализ.

### 6.4 Интеграционный чеклист

- Прогнать сценарии:
  - обычный диалог (2-3 реплики с памятью контекста);
  - `/interests`, `/criteria`, затем `/analyze`;
  - обычный текст после установки interests/criteria (без их передачи в prompt);
  - `/clear` и повторный чат без старого контекста;
  - невалидный YouTube URL;
  - видео без transcript (ожидаемая ошибка).

## Критерии приемки

- [ ] Нет ошибок отправки в Telegram из-за форматирования.
- [ ] Prompt-контракты соблюдаются для chat/analyze режимов.
- [ ] Основные интеграционные сценарии проходят локально.
