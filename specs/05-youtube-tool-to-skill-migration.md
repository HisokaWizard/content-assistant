# Spec 05: Миграция YouTube Tool в Skill

## Цель

Убрать псевдо-tool в markdown и зафиксировать анализ YouTube как skill-инструкцию для агента.

## Проблема

Файл `.opencode/tools/youtube.md` описывал алгоритм, но не являлся исполняемым tool-контрактом. Это вводило в заблуждение по архитектуре.

## Пошаговый план

### 5.1 Перенос в skill-структуру

- Удалить `.opencode/tools/youtube.md`.
- Создать `.opencode/skills/youtube-analysis/SKILL.md`.

### 5.2 Нормализация описания

- Описать явные:
  - входы (`youtubeUrl`, `criteria`, `interests`);
  - алгоритм;
  - формат ответа;
  - ограничения и fallback-поведение.

### 5.3 Синхронизация документации

- Убедиться, что в спеках и документации нет ссылок на “реализованный YouTube tool”, если он не является исполняемым компонентом.
- Зафиксировать, что это skill-поведение агента.

## Критерии приемки

- [ ] В репозитории нет `youtube.md` в `.opencode/tools`.
- [ ] Есть `SKILL.md` для YouTube-анализа.
- [ ] Поведение описано как skill, а не как runtime tool.
