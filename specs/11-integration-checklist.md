# Spec 11: Полный Ручной Тест (Telegram + OpenCode) Для Новичка

## Цель

Дать пошаговую инструкцию с нуля: как создать Telegram-бота, получить токен, запустить локально и проверить, что логика сессий работает правильно.

## Как Telegram-боты работают (простыми словами)

1. Ты пишешь сообщение боту в приложении Telegram.
2. Сообщение сначала попадает на серверы Telegram.
3. Твой код бота забирает новые сообщения через Bot API.
4. В этом проекте используется `long polling`: код сам регулярно спрашивает Telegram "есть ли новые сообщения?".
5. Когда сообщение получено, бот выполняет логику в `bot/src/index.ts`.
6. Дальше бот отправляет запросы в OpenCode (`/session`, `/session/:id/message`).
7. Ответ OpenCode бот отправляет обратно через Telegram API.
8. Ты видишь ответ в чате с ботом.

## Что нужно установить и проверить до Telegram

1. Убедиться, что OpenCode сервер запущен и доступен по `http://localhost:8888`.
2. Проверить API OpenCode: открыть `http://localhost:8888/doc`.
3. Проверить сборку бота:

```bash
cd bot
npm install
npm run build
```

## Шаг 1: Создать бота в Telegram и получить токен

1. Открыть Telegram.
2. Найти пользователя `@BotFather`.
3. Нажать `Start`.
4. Отправить команду:

```text
/newbot
```

5. Ввести имя бота (любое, например `Content Assistant Test`).
6. Ввести username бота (должен заканчиваться на `bot`, например `content_assistant_test_bot`).
7. BotFather отправит токен вида:

```text
1234567890:AA...
```

8. Скопировать токен и никому не передавать.

## Шаг 2: Где потом писать своему боту

1. В сообщении BotFather будет ссылка на твоего бота, например `@content_assistant_test_bot`.
2. Открой этого бота в Telegram.
3. Нажми `Start`.
4. Это и есть интерфейс: обычный чат, куда ты отправляешь команды и текст.

## Шаг 3: Запустить проект локально с токеном

1. Открыть терминал в проекте.
2. Перейти в папку бота:

```bash
cd /Users/hisokawizard/Projects/content-assistant/bot
```

3. Экспортировать переменные окружения:

```bash
export TELEGRAM_TOKEN="вставь_сюда_токен_из_BotFather"
export OPENCODE_URL="http://localhost:8888"
```

4. Запустить бота:

```bash
npm run dev
```

5. В терминале должно появиться:

```text
Bot started
```

## Шаг 4: Базовая проверка в Telegram

1. Отправить `/start`.
2. Отправить `/help`.
3. Отправить обычный текст: `Привет`.
4. Ожидается ответ модели.

## Шаг 5: Проверка логики сессий через Telegram и OpenCode API

### 5.1 Проверка chat-сессии

1. В Telegram отправить 2-3 обычных текстовых сообщения.
2. В отдельном терминале проверить список сессий:

```bash
curl -s http://localhost:8888/session
```

3. Ожидается одна chat-сессия (переиспользуется).

### 5.2 Проверка перехода в видео-flow

1. В Telegram отправить:

```text
/interests AI, автоматизация
```

2. Отправить:

```text
/criteria практичность, качество аргументов
```

3. Отправить YouTube ссылку:

```text
/analyze https://youtu.be/dQw4w9WgXcQ
```

4. Ожидаемое поведение:
   - старая chat-сессия закрывается;
   - создается новая video-сессия;
   - бот ждет около 3 секунд;
   - бот отправляет запрос анализа;
   - после ответа video-сессия удаляется.
5. Проверить сессии:

```bash
curl -s http://localhost:8888/session
```

6. Ожидается пустой список.

### 5.3 Проверка возврата в chat-flow

1. После видео отправить обычный текст.
2. Ожидается создание новой chat-сессии.
3. Проверить:

```bash
curl -s http://localhost:8888/session
```

4. Ожидается снова одна chat-сессия.

### 5.4 Проверка очистки

1. Отправить `/clear`.
2. Проверить сессии:

```bash
curl -s http://localhost:8888/session
```

3. Ожидается пустой список.

## Шаг 6: Ручной тест без Telegram (если токена пока нет)

### 6.1 Создать сессию

```bash
curl -s -X POST http://localhost:8888/session \
  -H "Content-Type: application/json" \
  -d '{"title":"manual-chat"}'
```

Сохранить `id` в `CHAT_ID`, подождать 3 секунды.

### 6.2 Отправить сообщение

```bash
curl -s -X POST http://localhost:8888/session/$CHAT_ID/message \
  -H "Content-Type: application/json" \
  -d '{"parts":[{"type":"text","text":"Привет, это ручной тест"}]}'
```

### 6.3 Удалить сессию

```bash
curl -s -X DELETE http://localhost:8888/session/$CHAT_ID
curl -s http://localhost:8888/session
```

Ожидается пустой список.

## Что делать, если не работает

1. Проверить, что OpenCode сервер поднят на нужном порту.
2. Проверить, что токен Telegram вставлен без лишних пробелов.
3. Проверить, что процесс `npm run dev` не остановлен.
4. Проверить ошибки в терминале бота.
5. Проверить доступность `http://localhost:8888/doc`.

## Критерии приемки

- [ ] `/start` и `/help` отвечают в Telegram.
- [ ] Обычный текст создает и переиспользует одну chat-сессию.
- [ ] Видео-запрос удаляет старые сессии, создает временную video-сессию и удаляет ее после ответа.
- [ ] После видео-сценария нет накопленных сессий.
- [ ] `/clear` очищает локальный контекст и сессии агента.
