# Spec 08: Single-Session Lifecycle

## Цель

Гарантировать, что у агента одновременно не более одной сессии.

## Правила жизненного цикла

### Текстовый чат

- При первом обычном текстовом сообщении:
  - удалить все существующие сессии (страховочный cleanup);
  - создать одну `chat`-сессию;
  - подождать 3 секунды;
  - отправить сообщение в созданную сессию.
- Для следующих текстовых сообщений использовать ту же `chat`-сессию.

### Видео-анализ

- При запросе на анализ видео:
  - очистить локальную историю;
  - удалить существующую `chat`-сессию и любые другие сессии;
  - создать новую `video`-сессию;
  - подождать 3 секунды;
  - отправить запрос анализа;
  - после ответа удалить `video`-сессию.

## Обработка ошибок

- Удаление `video`-сессии выполняется также в `finally` как best effort cleanup.

## Критерии приемки

- [ ] Перед видео-анализом очищаются все старые сессии.
- [ ] После видео-анализа сессии агента не остаются.
- [ ] При текстовом чате используется одна текущая `chat`-сессия.
