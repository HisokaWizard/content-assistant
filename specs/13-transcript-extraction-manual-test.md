# Spec 13: Ручной Тест Извлечения Transcript (yt-dlp)

## Цель

Проверить, что transcript извлекается стабильно и без участия LLM.

## Подготовка

1. Установить `yt-dlp`:
```bash
brew install yt-dlp
```
2. Проверить установку:
```bash
yt-dlp --version
```
3. Если YouTube возвращает anti-bot, добавить cookies-конфиг в `bot/.env` (любой один вариант):
```bash
YTDLP_COOKIES_FROM_BROWSER=chrome
```
или
```bash
YTDLP_COOKIES_FILE=/absolute/path/to/cookies.txt
```
4. Перезапустить бота после изменения `.env`.

## Сценарии

### 1) Публичное видео с субтитрами

1. Отправить боту `/analyze <url>`.
2. Ожидается успешный детальный анализ.
3. Проверить, что в ответе нет reasoning и служебных метаданных.
4. Если без cookies не работает, повторить с настроенными cookies.

### 2) Публичное видео без субтитров

1. Отправить боту `/analyze <url без captions>`.
2. Ожидается отказ в анализе с сообщением "без transcript анализ невозможен".

### 3) Длинное видео (1-2 часа)

1. Отправить боту `/analyze <длинное видео>`.
2. Ожидается структурированный ответ с:
   - 6-10 тезисами;
   - фактами/аргументами;
   - выводами и практическими шагами.
3. Ответ должен прийти одним сообщением Telegram.

## Технические проверки

1. Во время анализа проверить, что у агента не накапливаются лишние сессии:
```bash
curl -s http://localhost:8888/session
```
2. После завершения видео-анализа список сессий должен быть пуст.

## Критерии приемки

- [ ] Transcript достается автоматически на публичных видео с captions.
- [ ] При anti-bot блокировке после настройки cookies transcript успешно извлекается.
- [ ] Без transcript нет анализа.
- [ ] Качество результата выше простого пересказа заголовка.
- [ ] Ответ укладывается в одно сообщение.
