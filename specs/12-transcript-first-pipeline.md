# Spec 12: Transcript-First Pipeline (Без Анализа Без Транскрипта)

## Цель

Сделать анализ видео детерминированным: сначала получить transcript алгоритмически, и только потом отправлять текст в LLM.

## Принцип

- Нет transcript -> нет анализа.
- Запрещен `metadata-only` режим.
- LLM анализирует только фактический текст transcript (плюс metadata для контекста).

## Архитектура потока

1. Бот получает YouTube URL.
2. Модуль `transcript.ts` извлекает `videoId` и metadata.
3. Модуль пытается скачать субтитры через `yt-dlp` (`subs` + `auto-subs`) для публичного видео.
4. VTT/SRT парсится в чистый текст transcript.
5. Если transcript валиден, бот формирует prompt и отправляет в OpenCode.
6. Если transcript не найден или слишком короткий, бот возвращает понятную ошибку.

## Правила качества

- Анализ должен явно опираться на transcript.
- Запрещены домыслы, если факты не подтверждаются текстом.
- Для больших transcript допускается техническое ограничение по длине с явным указанием в prompt.

## Ошибки и fallback

- `yt-dlp` не установлен -> показать инструкцию установки.
- У видео нет субтитров -> сообщить, что анализ невозможен без transcript.
- Ошибки сети/доступа -> сообщить краткую техническую причину.

## Критерии приемки

- [ ] Видео анализируется только при наличии transcript.
- [ ] Без transcript возвращается отказ без псевдо-анализа.
- [ ] Ответ LLM содержит тезисы, факты, выводы и практические шаги.
