# План MVP - Content Assistant (Упрощенный)

## 1. Цель MVP

Максимально простая система для проверки гипотезы: анализ YouTube-видео через Telegram-бота с использованием агента opencode в режиме сервера.

## 2. Основные Принципы MVP

- Никакой базы данных — всё состояние хранится в сессии агента opencode.
- Никакого backend-сервиса — Telegram-бот общается напрямую с opencode agent API.
- Только бесплатная обработка YouTube-видео (transcript).
- Opencode Agent работает в режиме сервера и хранит состояние пользователя.
- Telegram бот — только UI для отправки команд и получения результатов.

## 3. Архитектура

```
Telegram User <-> Telegram Bot <-> Opencode Agent API (server mode) <-> YouTube Transcript
```

### 3.1 Telegram Бот

Зоны ответственности:
- принимать команды и сообщения;
- обнаруживать YouTube-ссылки;
- показывать прогресс;
- отображать результаты анализа.

Тонкий клиент — вся логика в opencode агенте.

### 3.2 Opencode Agent (Server Mode)

Зоны ответственности:
- хранить состояние пользователя (интересы, критерии, история);
- обрабатывать YouTube-ссылки (транскрипт → summary + тезисы + рекомендация);
- управлять сессией пользователя;
- применять простые лимиты (количество запросов в день).

Агент работает как сервер и хранит всё в своей сессии.

### 3.3 YouTube Processing

- Получить transcript через бесплатный API (youtube-transcript-cli или аналог).
- Определить язык.
- Если не русский — перевести.
- Сгенерировать summary + тезисы + рекомендацию через LLM.

## 4. Границы MVP

### Входит

- Telegram-бот с командами;
- анализ YouTube-видео через transcript;
- перевод на русский;
- генерация summary, тезисов, рекомендации;
- базовые команды управления (/start, /help, /analyze);
- хранение состояния в сессии агента.

### Не Входит (Отложено)

- Telegram-каналы и сводки;
- база данных;
- сложные лимиты и tracking;
- несколько сессий;
- модель выбора;
- аудио-ответ;
- веб-интерфейс.

## 5. Команды Бота (Минимальный Набор)

- `/start` — начать работу
- `/help` — показать доступные команды
- `/analyze <youtube_url>` — проанализировать видео
- `/interests <text>` — задать интересы
- `/criteria <text>` — задать критерии оценки видео
- `/clear` — очистить текущую сессию

Автоматическое обнаружение YouTube URL в любом сообщении.

## 6. Формат Результата

При анализе видео пользователь получает:

```
📹 <Название видео>
⏱ <Длительность>

📝 Краткое summary (2-3 предложения)

🔑 Ключевые тезисы:
1. ...
2. ...
3. ...

👍 Рекомендация: Смотреть / Не смотреть
📊 Оценка по критериям: X/10
```

## 7. План Реализации

### Этап 1. Настройка Opencode Agent

- Настроить opencode в режиме сервера.
- Определить промпт агента для обработки YouTube.
- Реализовать tool для получения transcript.
- Реализовать tool для генерации summary.

### Этап 2. Telegram Бот

- Создать Telegram-бота через BotFather.
- Настроить обработку команд.
- Интегрировать с opencode agent API.
- Реализовать обнаружение YouTube URL.

### Этап 3. Тестирование

- Протестрировать полное flow: ссылка → transcript → summary → ответ пользователю.
- Проверить работу с разными языками.
- Проверить лимиты.

## 8. Критерии Готовности MVP

- Пользователь может отправить YouTube-ссылку боту и получить summary с тезисами и рекомендацией.
- Бот работает без базы данных и backend-сервиса.
- Состояние пользователя сохраняется между запросами в сессии агента.
- Система запускается локально за 10-15 минут.

## 9. Следующий Шаг

После успешного MVP:
- Добавить Telegram-каналы (сводки);
- Добавить выбор модели;
- Добавить лимиты и tracking;
- Рассмотреть базу данных при необходимости.
